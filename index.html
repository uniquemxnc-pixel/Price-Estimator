Price Estimator (Spreadsheet-Driven)

Import each product line as its own spreadsheet. The file name becomes the product-line name.
Quantities appear on the right once you select products.
Subtotal: $0.00
Spreadsheet format (important)
The first row must be headers. Column A should be the product name (header can be Product). Each additional column is a buyer type (e.g., Retail, Wholesale, Distributor). Each row thereafter is a product, with numeric prices. Example: 
Product | Retail | Wholesale
Plate 10" | 12.50 | 9.25
Mug 12oz | 8.00 | 5.75
Where data is stored
Imported price sheets are stored locally in your browser (LocalStorage). This is intentionally offline-friendly. If you open this app on another device, import the sheets there as well (or host a shared backend later).
Status: No price sheets imported yet.

Quick start

  1. Click Download template and mirror the format in your spreadsheets.
  2. Import one spreadsheet per product line (file name becomes the line name).
  3. Select a line -> select buyer type -> pick products -> enter quantities.
  4. Use Copy quote to paste totals into a text/email.

Sharing as a URL

This is a single-file web app. To share it across devices/company:
  • Host the file on your internal SharePoint/Google Drive as a static webpage, or
  • Use GitHub Pages / Netlify / Vercel (free tiers) and share the link.
If you want centralized price data (no per-device imports), you will need a small backend (e.g., Google Sheets API, Airtable, Supabase).

Admin actions

Tip: export JSON once, then share that JSON file internally so sales reps can import everything in one click.
